## 目标检测的核心任务

目标检测（Object Detection）需要同时回答两个问题：

- **是什么**（分类 Classification）：图中有什么物体？
- **在哪里**（定位 Localization）：物体在什么位置？

输出通常是 `(类别, 置信度, x, y, w, h)` 的五元组。

过去十年，从两阶段到单阶段，从 Anchor-Based 到 Anchor-Free，目标检测经历了巨大的技术变革。

## 发展脉络总览

```
2014 ─ R-CNN ───────────────── 两阶段时代开启
2015 ─ Fast R-CNN ───────────── RoI Pooling 加速
2015 ─ Faster R-CNN ─────────── RPN 端到端训练
2016 ─ YOLOv1 ───────────────── 单阶段检测鼻祖
2016 ─ SSD ──────────────────── 多尺度特征检测
2017 ─ YOLOv2 / YOLO9000 ────── 更强更快更准
2018 ─ YOLOv3 ───────────────── FPN + 多尺度
2020 ─ YOLOv4 ───────────────── Bag of Freebies
2020 ─ YOLOv5 ───────────────── PyTorch 生态
2022 ─ YOLOv6 / YOLOv7 ──────── 工业级优化
2023 ─ YOLOv8 ───────────────── 全能统一框架
```

## 第一阶段：两阶段检测器

### R-CNN（2014）

核心思想：**先找候选区域，再分类**

```
输入图像
  │
  ▼
① 选择性搜索（Selective Search）→ 提取约 2000 个候选区域
  │
  ▼
② 每个区域缩放到固定大小 → CNN 提取特征
  │
  ▼
③ SVM 分类 + 边界框回归
```

**缺点**：每个候选区域都要跑一次 CNN，速度极慢（~13秒/图）。

### Fast R-CNN（2015）

```
输入图像 → CNN → 特征图 → RoI Pooling → 全连接层 → 分类 + 回归
```

改进：
- 整图只过一次 CNN，共享特征计算
- RoI Pooling 统一特征尺寸

### Faster R-CNN（2015）

**核心创新：RPN（Region Proposal Network）**

```
输入图像 → CNN → 特征图
                    │
        ┌───────────┴───────────┐
        ▼                       ▼
    RPN 网络                 RoI Pooling
  （生成候选区域）          （区域分类+回归）
```

RPN 替代了 Selective Search，实现了**端到端训练**，速度大幅提升。

## 第二阶段：单阶段检测器

### YOLOv1（2016）— You Only Look Once

**革命性思想：把检测问题变成回归问题**

```
输入图像（448×448）
    │
    ▼
CNN 直接输出 S×S 网格预测
    │
    ▼
每个网格预测 B 个边界框 + C 个类别概率
```

- 将图像划分为 7×7 网格
- 每个网格预测 2 个边界框 + 20 个类别
- 端到端训练，推理速度极快（45 FPS）

**优点**：快！背景误检率低
**缺点**：小目标检测差，定位精度不够

### YOLOv2 / YOLO9000（2017）

关键改进：

- **Batch Normalization**：加快收敛，提升 2% mAP
- **高分辨率分类器**：448×448 预训练
- **Anchor Boxes**：引入先验框机制
- **多尺度训练**：增强尺度鲁棒性
- **Darknet-19**：更轻量的骨干网络

### YOLOv3（2018）

```
骨干网络：Darknet-53（53 层卷积）

    特征金字塔 FPN
    ┌──────────────┐
    │  13×13 特征   │ → 大目标检测
    │  26×26 特征   │ → 中目标检测
    │  52×52 特征   │ → 小目标检测
    └──────────────┘
    
分类器：Sigmoid（支持多标签）
```

**核心突破**：FPN（特征金字塔网络）实现三个尺度的检测，大幅提升小目标精度。

### YOLOv4（2020）

总结了两大类优化技巧，系统化提升性能：

| 类别 | 代表技术 |
|------|---------|
| **Bag of Freebies**（不增加推理开销） | Mosaic 数据增强、CIoU Loss、CmBN |
| **Bag of Specials**（少量增加开销） | Mish 激活函数、SPP、PANet |

- 骨干网络：CSPDarknet53
- Neck：SPP + PANet（路径聚合网络）
- Head：YOLOv3 头部

### YOLOv5（2020）

最大的贡献是**工程化**：
- 完整的 PyTorch 实现
- 丰富的模型尺度：n / s / m / l / x
- 自动锚框计算
- 超参数进化

### YOLOv8（2023）— 当前版本

```
全新架构设计：
- Anchor-Free 检测头（告别手动设计锚框）
- C2f 模块（更高效的特征提取）
- 解耦检测头（分类和回归分支分离）
- TaskAlignedAssigner（智能正负样本匹配）

多任务支持：
- 目标检测（Detect）
- 实例分割（Segment）
- 图像分类（Classify）
- 姿态估计（Pose）
```

## 核心概念解析

### IoU（交并比）

```
IoU = 预测框 ∩ 真实框 / 预测框 ∪ 真实框

    ┌──────────┐
    │  ┌───────┼───┐
    │  │  交集   │   │
    └──┼───────┘   │
        └──────────┘
    
IoU = 0.75 → 好
IoU = 0.30 → 差
```

### mAP（平均精度均值）

目标检测最核心的评估指标：

```
mAP@0.5       → IoU 阈值 0.5 时的平均精度
mAP@0.5:0.95  → IoU 从 0.5 到 0.95（步长 0.05）的平均精度均值
```

### NMS（非极大值抑制）

消除重叠的检测框：

```
对同一类别的所有检测框：
1. 按置信度排序
2. 取最高分框，移除与它 IoU > 阈值（如 0.5）的其他框
3. 重复直到处理完所有框
```

## 选型建议

| 场景 | 推荐模型 | 理由 |
|------|---------|------|
| 实时检测（>30FPS） | YOLOv8-n / YOLOv5-n | 极快的推理速度 |
| 工业高精度 | YOLOv8-x / YOLOv7 | 最高的 mAP |
| 移动端部署 | YOLOv8-nano / NanoDet | 极小模型体积 |
| 研究实验 | YOLOv8 | 完整生态，易扩展 |
| 分割任务 | YOLOv8-seg | 原生实例分割支持 |

## 未来趋势

- **Transformer 检测器**：DETR、Deformable DETR、DINO
- **多模态大模型**：Grounding DINO（文本引导检测）
- **Zero-shot 检测**：OWL-ViT、SAM
- **边缘计算**：TinyML、模型蒸馏、量化部署

## 总结

目标检测技术在过去十年经历了从两阶段到单阶段、从 Anchor-Based 到 Anchor-Free 的范式转变。YOLO 系列以其卓越的速度 - 精度平衡，依然是工业场景的首选方案。

理解模型演进，比会用某个模型更重要——因为你可以更好地根据实际需求选型、调优和创新。
